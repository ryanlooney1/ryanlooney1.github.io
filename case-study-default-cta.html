Improving the default call to action

Redesigning a core CTA pattern to improve clarity, strengthen decision-making, and create a more scalable product experience.

Role: Content Design Lead Scope: Core product UX Focus: Clarity, conversion, systems thinking

Overview
The challenge

A default CTA pattern was creating friction for users and limiting the product’s ability to guide action clearly and consistently.

My role

I led the content strategy for the change, helped define the new pattern, and aligned cross-functional partners around a clearer direction.

The outcome

The redesigned CTA experience made key actions easier to understand, improved usability, and created a stronger foundation for scaling the pattern.

The challenge

Calls to action often seem simple, but they carry a lot of responsibility. In this case, the default pattern was not doing enough to guide users toward the right next step. The language was serviceable, but not strong enough to support a high-quality product decision at scale.

Key insight

Effective CTAs do more than label a button. They clarify intent, reduce hesitation, and help the interface feel more purposeful and trustworthy.

Approach
1. Evaluated the existing pattern

I identified where the default CTA was creating ambiguity and where the language was falling short of the experience the product needed.

2. Clarified the decision model

I worked to define what action the interface was really asking users to take and how the CTA could better support that moment.

3. Designed for reuse

I partnered across disciplines to create a clearer CTA pattern that could scale beyond one screen or use case and remain consistent over time.

Impact

This redesign made a key action easier to understand and created a reusable pattern the product could apply more consistently over time.

  • Improved clarity and confidence around the primary user action
  • Created a stronger CTA pattern that could scale across future flows
What this project shows

Small interface moments can have outsized impact

This project highlights how I approach seemingly simple UX problems: identify the hidden complexity, clarify the user decision, and create a content solution that improves both the experience in front of the user and the system behind it.
